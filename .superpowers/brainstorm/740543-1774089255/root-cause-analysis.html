K线图缩放Bug分析

仅在1Week、1Month、All时间范围出现

问题特征

  • 触发条件：仅在 1Week (1W)、1Month (1M)、All (1Y) 时间范围
  • 触发操作：缩放太多 或 向左移动太多
  • 表现：K线消失，变成贴合X轴的一条线
  • 你的假设：可能是超出了数据范围
问题场景示意
✓ 正常范围
用户在有数据的时间范围内查看
数据: 2023-03-22 ~ 2026-03-22
视图: 2025-01-01 ~ 2026-03-22
结果: K线正常显示 ✓
✗ 超出范围
用户缩放/左移到没有数据的区域
数据: 2023-03-22 ~ 2026-03-22
视图: 2020-01-01 ~ 2021-12-31
结果: 变成一条线 ✗
为什么只在1W/1M/All出现？
这些时间范围跨度大（3年/5年/10年），用户更容易缩放或左移到数据范围之外。
而15M/1H/4H/1D这些短期范围，数据通常是连续的，不容易超出。

可能的根本原因

让我们一起分析几种可能性：

A

数据范围不足

数据库中crypto数据不够长（比如只有3年），但用户可以缩放到10年前

支持证据：
  • 只在长时间范围出现
  • 向左移动触发（查看更早数据）
B

图表渲染问题

lightweight-charts在没有数据的区域渲染异常，显示为一条线

支持证据：
  • 变成"一条线"是视觉渲染问题
  • 可能是库的默认行为
C

Y轴缩放问题

当视图范围内数据点很少或没有时，Y轴自动缩放出现问题，导致K线被压扁

支持证据：
  • "贴合X轴"说明高度被压缩
  • 可能是价格范围计算错误
D

时间格式转换问题

1W/1M/All使用日期格式(YYYY-MM-DD)，超出范围时时间轴处理异常

支持证据：
  • 代码中intraday用timestamp，daily用date string
  • 可能是时间格式边界处理问题

💡 下一步

为了确定根本原因，我们需要：

  1. 检查数据库中crypto数据的实际时间范围
  2. 查看lightweight-charts在空数据区域的行为
  3. 检查Y轴自动缩放的配置
  4. 测试边界情况
